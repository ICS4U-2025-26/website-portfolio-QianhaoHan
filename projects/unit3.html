This unit explores algorithmic thinking, recursion, and introduces machine learning and artificial intelligence concepts through hands-on coding activities.

Activity: Recursion Applications - Fractal Drawings

Task Description

Explore recursion by implementing fractal drawing algorithms using Python's Turtle graphics library. The activity involved creating two classic fractals: a recursive binary tree and a Sierpinski triangle, both demonstrating how a function can call itself to produce complex, self-similar patterns from simple rules.

My Approach

Since our teacher gave us the code to study, my approach was to focus on understanding how the recursion worked in each function. I looked at how both examples used a base case and a recursive step. The binary tree showed how the turtle draws branches and then backtracks to the correct position, while the Sierpinski triangle showed how one shape can repeat in smaller versions. Studying the code helped me understand how recursion creates self-similar patterns.

Code Snippet - Binary Tree

def draw_tree(t, branch_len, scale, angle):
    """
    Recursive function to draw a binary tree.
    
    Args:
        t: The turtle object
        branch_len: Current length of the branch
        scale: How much the length shrinks each step (e.g. 0.75)
        angle: The angle of the split
    """
    # BASE CASE: Stop when branches are too small
    if branch_len < 10:
        t.color("green")
        t.forward(branch_len)
        t.backward(branch_len)
        t.color("white")
        return

    # Draw the trunk/branch
    t.pensize(branch_len / 10)
    t.forward(branch_len)
    
    # RECURSIVE STEP: Right Branch
    t.right(angle)
    draw_tree(t, branch_len * scale, scale, angle)
    
    # RECURSIVE STEP: Left Branch
    t.left(angle * 2)
    draw_tree(t, branch_len * scale, scale, angle)
    
    # RESTORE STATE (Backtracking)
    t.right(angle)
    t.backward(branch_len)

Code Snippet - Sierpinski Triangle

def draw_sierpinski(t, order, size):
    """
    Draws a Sierpinski Triangle.
    order: Depth of recursion
    size: Length of the triangle side
    """
    if order == 0:
        # BASE CASE: Draw a single triangle
        for _ in range(3):
            t.forward(size)
            t.left(120)
    else:
        # RECURSIVE STEP: Draw 3 smaller triangles
        draw_sierpinski(t, order - 1, size / 2)
        t.forward(size / 2)
        draw_sierpinski(t, order - 1, size / 2)
        t.backward(size / 2)
        t.left(60)
        t.forward(size / 2)
        t.right(60)
        draw_sierpinski(t, order - 1, size / 2)
        t.left(60)
        t.backward(size / 2)
        t.right(60)

What I Learned

This activity helped me better understand recursion in a visual way. I was already somewhat familiar with recursion, but this was my first time learning about fractals. I learned how important the base case and recursive step are, and I also saw how backtracking helps the turtle return to the right place. Overall, I learned that fractals are created by repeating the same pattern at smaller and smaller scales.
