Unit 6: Bobcats' Den Final Project

For my final project, my team created MediaWise, a media literacy platform with an educational website and a Chrome extension that helps users think critically about misinformation, bias, and unreliable online content.

Final Product: MediaWise

MediaWise was designed to help users, especially high school students, recognize misinformation and evaluate online content more carefully. Instead of simply telling users whether something is true or false, the project encourages users to ask better questions about source credibility, bias, emotional language, and evidence quality.

Problem

Online misinformation and AI-generated media make it harder for students to know which sources are reliable. MediaWise helps users build media literacy habits while reading online content.

Target Audience

The main audience is high school students ages 14–18. A secondary audience is older adults who may also benefit from a simple tool for evaluating digital information.

Theme Connection

The project connects to technology and app ideas by using a website and browser extension to support media literacy and responsible technology use.

My Main Role

I led the project and was mainly responsible for the Chrome extension and backend server, including the OpenAI API connection, selected-text analysis, chatbot follow-ups, and fallback analysis logic.

Technical Architecture

The website was built with HTML, CSS, and JavaScript. HTML organizes the pages and learning content, CSS creates the visual design, and JavaScript powers interactive features such as the quiz and result feedback.

The Chrome extension uses a Manifest V3 structure with separate files for the popup interface, content scripts, styling, and backend server. JavaScript collects highlighted text and page metadata, while a Node.js and Express backend sends text to the OpenAI API through routes such as /analyze and /chat. This keeps the API key private instead of storing it directly in the extension.

Advanced Programming Concepts

  • Modular design: The website, extension popup, content script, backend server, and styling are separated into multiple files and folders.
  • Algorithms: The quiz checks answers, calculates scores, and gives feedback. The extension organizes risk level, category scores, loaded words, and verification questions.
  • Fallback logic: If the AI request fails, the extension uses a rule-based fallback that scans for loaded words and weak evidence phrases.
  • Complex data structures: Objects and arrays store selected text, page metadata, chat history, category scores, loaded words, and AI responses.
  • File input/output and configuration: Files such as manifest.json, package.json, and .env organize extension settings, dependencies, and API key storage.

Ethics, Sustainability, and Emerging Technology

Because MediaWise uses AI, one ethical concern is that users might trust the AI too much. To address this, the extension is designed to guide users through critical thinking rather than replacing their own judgment. It highlights possible bias, emotional language, evidence quality, source credibility, and verification questions.

We also considered sustainability by keeping requests intentional and focused. The extension analyzes selected or pasted text instead of constantly scanning entire webpages in the background, which keeps AI requests smaller and more efficient.

The project connects to emerging computer science topics such as natural language processing, misinformation detection, AI-generated media, and responsible AI.
